# Rails Application

A Ruby on Rails 8.1.1 application running on Ruby 3.4.7, configured to run with Docker.

## Prerequisites

- Docker
- Docker Compose

## Getting Started

### 1. Build and start the containers

```bash
docker compose build
docker compose up
```

Or run in detached mode:

```bash
docker compose up -d
```

### 2. Set up the database

In a new terminal, run:

```bash
docker compose run web rails db:create
docker compose run web rails db:migrate
```

### 3. Access the application

Open your browser and navigate to: http://localhost:3000

## Common Commands

### Run Rails console

```bash
docker compose run web rails console
```

### Run database migrations

```bash
docker compose run web rails db:migrate
```

### Run tests

```bash
docker compose run web rails test
```

### Install new gems

After adding a gem to `Gemfile`:

```bash
docker compose run web bundle install
docker compose build
```

### Stop the containers

```bash
docker compose down
```

### Stop and remove volumes (clears database)

```bash
docker compose down -v
```

## Project Structure

- `Dockerfile.dev` - Development Docker configuration
- `Dockerfile` - Production Docker configuration (generated by Rails)
- `docker-compose.yml` - Docker Compose configuration for development
- `config/database.yml` - Database configuration (configured for Docker)

## Technology Stack

- **Ruby**: 3.4.7
- **Rails**: 8.1.1
- **Database**: PostgreSQL 16
- **Web Server**: Puma

## Development

The application is configured to run in development mode with:
- Hot reloading enabled
- Volume mounts for live code changes
- PostgreSQL database service
- All dependencies containerized
